## Getting Started

### What is Dream Team Hub?

Dream Team Hub is your comprehensive multi-pod orchestration platform designed to streamline complex organizational workflows. It unifies management across distributed teams with:

- **21 Organizational Pods** with unique color-coded identities
- **58 Fully-Specified Agents** with complete skill packs
- **AI-Powered Chat** with 32 role-based personas
- **Complete Project Management** with tasks, files, and messaging
- **Automated Backups** via nightly snapshots
- **Professional UI** with dark glass-morphism design

### System Requirements

- **Modern web browser** (Chrome, Firefox, Safari, Edge)
- **Internet connection**
- **Replit account** (for authentication)

### Logging In

1. **Navigate to Dream Team Hub** in your browser
2. **Click "Login with Replit"** button
3. **Choose your authentication method:**
   - Google Account
   - GitHub Account
   - Apple ID
   - X (Twitter)
   - Email/Password

4. **Grant permissions** when prompted
5. **You're in!** You'll be redirected to the Dashboard

## Agent System (Role Cards)

![Agents Page Screenshot](58 agent cards with colored pod rails)

### What are Agents?

**Agents** (also called Role Cards) are your Dream Team members - each representing a specific role with:
- **Expertise areas** and skill packs
- **Pod assignment** (shown via colored rail)
- **RACI responsibilities** (Responsible, Accountable, Consulted, Informed)
- **Communication style** and personality
- **AI persona** for Dream Team Chat

### Viewing Agents

The Agents page displays all **58 agents** as cards:

**Visual Elements:**
- **Colored Rail** (left side) - Shows pod affiliation
- **Agent Avatar** - Profile image or initials
- **Title** - Role name (e.g., "Aegis - Security Guardian")
- **Handle** - Unique identifier (e.g., @aegis)
- **Pod Badge** - Which pod they belong to
- **Skill Tags** - Key expertise areas

### Agent Details

**Click any agent card** to see full profile:

1. **Identity**
   - Title and Handle
   - Pod assignment
   - Avatar/Photo

2. **Expertise**
   - Primary skills
   - Secondary skills
   - Certifications

3. **RACI Matrix**
   - Responsible for
   - Accountable for
   - Consulted on
   - Informed about

4. **Agent Specifications**
   - Detailed skill packs
   - Tools & technologies
   - Communication preferences
   - Personality traits

5. **Activity**
   - Recent conversations
   - Current assignments
   - Work items

### Creating a New Agent

1. Click **"+ New Agent"** button
2. Fill in the form:

**Required Fields:**
- **Title** - Role name (e.g., "Data Wizard")
- **Handle** - Unique @handle (e.g., @datawiz)
- **Pod** - Select from dropdown (affects color coding)

**Optional Fields:**
- **Expertise Areas** - Skills and specializations
- **RACI Roles** - Define responsibilities
- **Communication Style** - How they interact
- **Avatar** - Upload profile image

3. Click **"Create Agent"**
4. **New agent appears** with pod-colored rail

### Agent Filtering & Search

**Filter by Pod:**
- Click pod color chips at top of page
- See only agents from that pod

**Search Agents:**
- Use search bar to find by name, handle, or skill
- Type to filter results in real-time

**Sort Options:**
- By Name (A-Z)
- By Pod
- By Recently Active

## Advanced Features

### 1. Manifest Importer (Bulk Pod Import)

**Use Case:** Import multiple pods at once from a manifest file

**How to Use:**

This is an API-based feature for administrators. Using a tool like Postman or curl:

**Endpoint:** `POST /api/import/new-pods`

**Request Body:**
```json
{
  "pods": [
    {
      "name": "New Pod Name",
      "charter": "Pod mission statement",
      "purpose": "Why this pod exists",
      "pillar": "Innovation",
      "type": "New",
      "priority": "High",
      "autonomyLevel": "L2"
    },
    {
      "name": "Another Pod",
      "charter": "Another mission",
      "purpose": "Another purpose",
      "pillar": "Impact"
    }
  ]
}
```

**Response:**
```json
{
  "success": true,
  "message": "Import completed: 2 created, 0 updated, 0 failed",
  "results": {
    "total": 2,
    "created": 2,
    "updated": 0,
    "failed": 0,
    "errors": []
  }
}
```

**Features:**
- ‚úÖ **Bulk creation** - Import many pods at once
- ‚úÖ **Smart updates** - Existing pods (by name) are updated
- ‚úÖ **Error handling** - Individual failures don't break entire import
- ‚úÖ **Validation** - Schema validation ensures data integrity

### 2. Agent Goldens (Nightly Snapshots)

**Automated Backup System** for all agent data

**How It Works:**
- **Automatic:** Runs every night at 2:00 AM
- **Complete:** Captures all 58 agents + 40 agent specs
- **Queryable:** Access historical snapshots
- **Safe:** Validates payload size (<50 MB)

**Manual Snapshot:**

**Endpoint:** `POST /api/agent-goldens/snapshot`

**Response:**
```json
{
  "success": true,
  "message": "Created snapshot of 58 agents and 40 specs",
  "golden": {
    "id": 1,
    "snapshotDate": "2025-11-03T05:00:00.000Z",
    "agentCount": 58,
    "metadata": {
      "triggeredBy": "manual",
      "duration": 145,
      "payloadSizeMB": 2.34,
      "checksum": "58-40-1730606400000"
    }
  }
}
```

**View Snapshots:**

**Endpoint:** `GET /api/agent-goldens?limit=30`

Returns list of recent snapshots with counts and dates.

**Get Specific Snapshot:**

**Endpoint:** `GET /api/agent-goldens/:id`

Returns full snapshot with complete agent and spec data.

**Use Cases:**
- **Audit trail** - Track changes over time
- **Rollback** - Restore previous agent configurations
- **Compliance** - Maintain historical records
- **Analysis** - Compare agent evolution

2. **Work:** Pod B completes the work
3. **Mirror Back:** Pod B reports outcome
   - Share results, links, decisions
4. **Log:** Automatically creates Decision Log entry

**Benefits:**
- ‚úÖ Clear communication
