Here’s a **paste-ready Replit drop** to add **“Convert to Work Item”** from a Strategy Session.

### Behavior

* Button appears on **Strategy Detail** page.
* Enabled only when `status === "LOCKED"` (keeps the “no execution without lock” rule real).
* Creates a Work Item via `POST /api/work-items` with a **minimal, tolerant payload**:

  * `title`
  * `inputs` (the strategy markdown body, so the thinking carries over)
  * `targetContext` (includes `strategySessionId` + repo hint so Work Tower chips show provenance)
* On success: redirects to `/work-items/:id` and shows a toast.
* On failure: shows toast with the HTTP status/message (no silent failure).
