# Default vs Pro UX (the simplest mental model)

### Default user should experience this page as:

**Templates Library (Work Orders)**

* “These are runbooks you can run. Active work is in Work Items.”
* Only show:

  * search
  * list of templates
  * a single “Run” CTA per template
  * minimal metadata

### Pro mode adds:

* “Advanced template details” (caps/KPIs/playbookHandle/stop)
* “Runs” table and debug info
* Anything “ops-y” stays behind the advanced panel

The key: **Default doesn’t show ops-looking data** (runs/cost/ms) because that screams “active work”.
